Conftest

A one-page card to revisit. Compressed essence of Lesson 5 — the convention, the CLI, the exit codes.

The one-sentence model

Conftest is OPA for config files: it parses a file into input, runs the rules named deny/warn/violation in package main, and turns the findings into a PASS/FAIL exit code your CI reads.
config file
YAML · Dockerfile · JSON…
→
conftest test
parse → input, run data.main.deny
→
exit code
0 pass · 1 fail

The rule convention

RuleMessageBuild effectExit
denystringfails1
violationstructured objectfails1
warnstringadvisory, passes0

Rules live in package main by default · optional underscore suffix for readability (deny_privileged) is ignored when matching · a rule is the same partial set rule you already write.

Canonical policy shape

package main

deny_privileged contains msg if {
	some c in input.spec.containers   # paths match the parsed file's shape
	c.securityContext.privileged == true
	msg := sprintf("%q runs privileged", [c.name])
}

warn_latest_tag contains msg if {
	some c in input.spec.containers
	endswith(c.image, ":latest")
	msg := sprintf("%q uses the mutable :latest tag", [c.name])
}

Directory layout

.
├── pod.yaml            # the config to check (becomes input)
└── policy/             # default policy dir (override with -p)
    ├── deny.rego       # package main
    └── warn.rego       # package main

CLI cheat sheet

# test a file against ./policy
conftest test pod.yaml
conftest test pod.yaml -p policy/          # explicit policy dir
conftest test deploy/*.yaml                # many files at once
cat pod.yaml | conftest test -             # from stdin

# machine-readable output for CI
conftest test pod.yaml -o json             # also: tap, table, junit, github

# namespaces (package other than main)
conftest test pod.yaml --namespace kubernetes
conftest test pod.yaml --all-namespaces

# check the gate in a script
conftest test pod.yaml; echo $?           # 0 = pass, 1 = a deny fired

# unit-test the policies themselves (test_ rules, no input file)
conftest verify -p policy/                 # 0 = all pass, 1 = a test failed

Reading the output

FAIL - pod.yaml - main - "api" runs privileged
WARN - pod.yaml - main - "web" sets no securityContext

3 tests, 2 passed, 1 warning, 0 failures, 0 exceptions
FAIL / WARN
One line per finding — the string your deny/warn set produced.
tests
Total rule evaluations, not files. Passed = rules that produced no finding.
failures
Only these (and violations) make the exit code non-zero. Warnings never do.
It's just OPA conftest test pod.yaml -o json shows the query it runs: data.main.deny. Anything you can do in opa eval — some, every, comprehensions, sprintf, helper rules — works unchanged here.

Supported input formats (selection)

YAML · JSON · Dockerfile · HCL / HCL2 (Terraform) · TOML · INI · XML · Jsonnet · CUE · Cyclonedx · Spdx · properties · edn · VCL · and more — chosen by file extension. Each becomes an input document you reach into by its own shape.
