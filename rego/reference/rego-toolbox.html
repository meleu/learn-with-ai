Minimal policy anatomy

package hello                     # namespace → data.hello

greeting := "Hello, Rego!"        # constant rule

allow if input.role == "admin"   # boolean rule; undefined when body fails

deny contains msg if {            # set rule: collects messages (conftest style)
    input.kind == "Deployment"
    msg := "some violation text"
}

Key facts

  • input = the JSON document under judgment; data = policies + external data.
  • Rule body lines are ANDed. Multiple definitions of the same rule = OR.
  • A boolean rule whose body fails is undefined, not false (use default allow := false to get a real false).
  • OPA ≥ 1.0: if and contains keywords are mandatory. Distrust older tutorials.
  • conftest convention: package main, rules named deny/warn/violation (suffixes allowed: deny_no_root).
