Iteration & Strings

A one-page card to revisit. Compressed essence of Lessons 3–4.

The one-sentence model

When config is a list, one rule iterates it: some … in binds a variable to each element and runs the body once per binding. With contains that gives one finding per matching item.

Iterate & collect findings — some

package manifest

violations contains msg if {
	some c in input.containers   # bind c to each element
	endswith(c.image, ":latest")   # condition (AND)
	msg := sprintf("container %q uses :latest", [c.name])
}

Body runs once per element · only bindings whose body holds contribute · findings union into one set · empty list ⇒ no findings ⇒ [].

All must pass? — every

default all_pinned := false
all_pinned if {
	every c in input.containers {
		not endswith(c.image, ":latest")
	}
}

One boolean · fails if any element breaks the body · needs a default · empty list ⇒ vacuously true (guard with count(xs) > 0).

Build a string — sprintf

sprintf("container %q on image %q", [c.name, c.image])
# → container "cache" on image "redis:latest"

Args go in an array, one slot per verb, left to right.

VerbFormatsExample → output
%vany value, default form[1,2] → [1, 2]
%sstring, unquotedapi → api
%qstring, quotedapi → "api"
%dinteger18 → 18
%ffloat1.5 → 1.500000
%%a literal percent→ %

Comprehensions (build a collection)

# set: { value | body }   — dedupes, unordered
names := {c.name | some c in input.containers}

# array: [ value | body ]  — keeps duplicates & order
images := [c.image | some c in input.containers]

# object: { key: value | body }  — keys must be unique
by_name := {c.name: c.image | some c in input.containers}

Read as "the collection of value for every binding where body holds." See Lesson 4.

FormHeadKeeps duplicates?Ordered?
set{v | …}nono
array[v | …]yesyes
object{k: v | …}keys uniqueno
Distinct-values idiom: count([x | …]) == count({x | …}) — array keeps dups, set drops them, so equal counts ⇒ no duplicates. And a comprehension never goes undefined: no matches ⇒ an empty collection, so guard with count(…) > 0 when empty shouldn't count.
Legacy iteration you'll see in the wild Older policies iterate with input.containers[_] or input.containers[i] instead of some … in. Both still work; prefer some x in xs — it names the element and reads clearly.

Handy iteration built-ins

count(xs)
Number of elements — fold a finding set to a verdict, or guard an empty list.
endswith / startswith / contains
Substring tests on strings (contains(s, sub) is the string builtin, distinct from the contains keyword).
sprintf(fmt, [args])
Format a message; verbs above.
some x in xs
Iterate elements. Also some k, v in obj for objects.
