Rego for Pipeline Policy-as-Code

A hands-on course for validating microservice configs with conftest & OPA.

Mission Author and maintain Rego policies (via conftest) that validate hundreds of microservices in your pipelines — and be fluent enough to teach your coworkers. See MISSION.md.

Lessons

  1. The Rego Mental Model — why Rego exists, what a .rego file really is (pure Rego + opa eval).
  2. Partial Rules: a Set of Findings — contains rules that collect many findings; the shape conftest's deny is built from.
  3. Iterating Over Config — some … in and every over lists; detailed messages with sprintf.
  4. Comprehensions — collect values, not just verdicts: set / array / object {v | …}, and using them inside a rule.
  5. Conftest: just OPA underneath — run your Rego on a real config file; deny/warn/violation and the exit code that gates the build.

Reference

  • Rego Mental Model & Toolbox — one-page card: glossary, conftest convention, CLI cheat sheet.
  • Complete vs Partial Rules — one-page card: the two rule shapes side by side, and how each prints.
  • Iteration & Strings — one-page card: some/every, comprehensions, and sprintf verbs.
  • Conftest — one-page card: the deny/warn/violation convention, CLI cheat sheet, and exit codes.
