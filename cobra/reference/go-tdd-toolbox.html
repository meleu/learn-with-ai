Go TDD & CLI Toolbox

The compressed essentials: the red-green-refactor loop, the test commands, table tests, math/rand/v2, and the clig.dev rules this course holds to. Built to print.

The TDD loop

1 · RED
Write one small failing test. Run it. Watch it fail (or fail to compile).
→
2 · GREEN
Write the least code that makes it pass. Nothing more.
→
3 · REFACTOR
Improve names/shape with tests green. Then loop.

Rule: never write production code except to make a failing test pass. A compile error (undefined: Foo) counts as a legitimate red.

The commands

CommandWhat it does
go mod init <path>Start a module (e.g. github.com/you/rndm). Creates go.mod.
go test ./...Run every test in the module. Pass = ok, fail = FAIL.
go test -vVerbose: prints each === RUN / --- PASS line.
go test -run TestNameRun only tests matching a regexp.
go build -o rndm .Compile the package into a binary named rndm.
go run .Compile + run in one step (no binary left behind).
go vet ./...Static checks for suspicious code. Keep it clean.
gofmt -l .Lists unformatted files. Empty output = all formatted.

Test anatomy

// file: rndm_test.go — must end in _test.go
package main

import "testing"

// func name starts with Test, takes *testing.T
func TestNumberInRange(t *testing.T) {
	got := Number()
	if got < 0 || got > 32767 {
		t.Errorf("Number() = %d; want [0,32767]", got)
	}
}
t.Errorf
Mark failed, keep going. Use for independent checks.
t.Fatalf
Mark failed, stop this test now. Use when continuing is pointless.
t.Helper()
Call inside a helper so failures report the caller's line.

Table-driven tests

The idiomatic Go pattern — one test, many cases, each an isolated subtest via t.Run:

func TestClamp(t *testing.T) {
	cases := []struct {
		name     string
		in, want int
	}{
		{"below", -5, 0},
		{"inside", 3, 3},
		{"above", 99, 10},
	}
	for _, c := range cases {
		t.Run(c.name, func(t *testing.T) {
			if got := Clamp(c.in); got != c.want {
				t.Errorf("Clamp(%d) = %d; want %d", c.in, got, c.want)
			}
		})
	}
}

Testing randomness

You cannot assert a random value equals a constant. Two honest strategies:

StrategyHow
Test a propertyDraw many times; assert every draw is in range / non-negative / etc.
Inject the sourcePass a seeded *rand.Rand in tests so the sequence is fixed and exactly assertable (later lesson).

math/rand/v2 quick ref

CallResult
rand.IntN(n)int in [0, n). Panics if n ≤ 0.
rand.N(n)Generic version for any integer type.
rand.Float64()float64 in [0.0, 1.0).
rand.Shuffle(n, swap)Shuffle n items via a swap func.
rand.New(rand.NewPCG(a, b))A seeded *rand.Rand → reproducible sequence.
v2 vs v1 Top-level funcs in math/rand/v2 are auto-seeded (different each run) and safe for concurrent use — no rand.Seed(...) call, which v2 removed. Seed explicitly only when you want reproducibility, via a Source (PCG/ChaCha8).

clig.dev rules we hold to

stdout
The program's actual output (the number). Clean, one value per line, pipeable.
stderr
Errors, logs, usage, diagnostics — everything that isn't the result.
exit 0
Success. Non-zero on failure so scripts can branch on it.
compose
“Expect the output of every program to become the input to another.”
humans first
Readable by a person, still parseable by a machine.
