Go CLIs Course

Building CLI tools in Go, test-first

Small, hands-on projects that grow from the standard library up to Cobra — every feature driven by a failing test, every UX call grounded in clig.dev. Current project: rndm.

The mission Ship real command-line tools your coworkers install and run — robust Go binaries, not fragile shell scripts — while deepening genuine fluency with Go's standard library. TDD is the working discipline throughout.

Lessons

  • Lesson 1 · rndm Test-drive rndm: red → green → refactor Drive a function from a failing test. Test randomness by asserting a property, not a value. Ship a binary that prints to stdout and exits 0.
  • Lesson 2 · rndm Testing the program's output (the io.Writer seam) Extract run(w io.Writer) so stdout is capturable in a test with bytes.Buffer — the classic move that makes a whole CLI testable. (coming next)
  • Lesson 3 · rndm Reproducible randomness: injecting the source Pass a seeded *rand.Rand so a test can assert an exact sequence — dependency injection / seams. (planned)
  • Lesson 4 · rndm Your first flag with stdlib flag Add --max / -n with the standard library — usage on stderr, exit codes — before reaching for Cobra. (planned)
  • Lesson 5+ · Cobra When flags outgrow flag: enter Cobra Introduce Cobra once the tool genuinely needs subcommands/flags, and migrate rndm gain-by-gain. (planned)
